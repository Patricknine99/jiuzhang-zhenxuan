# 九章甄选 — 业务拓展路线图（idea2）

> 记录时间：2026-05-02
> 执行优先级：从上到下

---

## 1. 需求方工作台深化

**目标**：已登录的需求方可以查看自己的需求列表、跟进状态、匹配到的服务商短名单。

**子任务**：
- 改造 `/buyers` 页，登录后展示个人需求面板（当前提交 + 历史提交）
- 需求列表按状态分组（待评估 / 匹配中 / 已交付 / 异常）
- 已匹配的服务商卡片 + AI 匹配理由摘要
- 新增 `/buyers/dashboard` 作为需求方首页入口

**数据库关联**：从 `leads` 表按 `type='demand'` 查询，后端新增 `/api/leads/mine` 接口

---

## 2. 供给方工作台深化

**目标**：已登录的供给方可以管理入驻资料、查看被匹配到的需求、接单状态。

**子任务**：
- 改造 `/creators` 页，登录后展示供给方面板（入驻状态 + 接单机会）
- 入驻申请进度（审核中 / 已认证 / 待补充材料）
- 匹配到的新需求列表（按适配度排序）
- 接单状态管理（已接单 / 进行中 / 待验收）

**数据库关联**：从 `leads` 表按 `type='application'` 查询入驻，后端新增 `/api/providers/matches` 接口

---

## 3. 管理员后台接真实数据

**目标**：当前 `/admin` 只读本地 localStorage，改造为接入 PostgreSQL 真实线索和审计日志。

**子任务**：
- 后端新增 `/api/admin/leads` 接口，从 `leads` 表读取全量线索
- 后端新增 `/api/admin/audit` 接口，从 `audit_logs` 表读取审计记录
- 前端 `/admin` 展示真实线索队列（按类型/状态筛选）
- 审计日志展示最近操作记录

**数据库关联**：leads 表 + audit_logs 表

---

## 4. 支付/订单闭环

**目标**：微信支付预下单从 503/501 占位推进到真实可用。

**前置条件**：
- 微信商户号、API v3 密钥、证书序列号
- Nginx 回调端点配置

**子任务**：
- 实现 JSAPI 统一下单签名
- 实现支付回调验签
- 订单表设计与创建
- 前端 `/status` 展示支付流水

---

## 5. 消息通知系统

**目标**：需求有新匹配、入驻有审核结果时，通过站内信 + 短信/邮件通知。

**子任务**：
- 通知表设计
- 站内信组件（右上角红点 + 消息列表）
- 短信/邮件渠道预留（接入云通信服务）

---

## 6. 服务商评级/评价

**目标**：交易完成后需求方可以给供给方打分评价，沉淀信用数据。

**子任务**：
- 评价表设计（评分 + 评价文本 + 标签）
- `/providers/[slug]` 页展示真实评价
- 需求方在完成交易后提交评价
- 供给方信用分累计
