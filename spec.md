# AI 创作者商业撮合平台 — 项目 Spec

> 最后更新：2026-05-01  
> 状态：策划阶段，准备出 Demo 首页

---

## 1. 一句话定位

**帮助企业、品牌方、中小商家发现、评估并对接经过真实商业验证的 AI 创作者与独立工作室。**

---

## 2. 核心定位

- **不是** AI 艺术作品社区、工具导航站、泛外包平台
- **更像** AI 版 Product Hunt + 站酷服务商 + 精选猪八戒 + 企业 AI 采购目录
- **核心壁垒**：跨工具、跨平台、面向企业采购的 AI 服务商精选撮合平台（当前市场空白）

---

## 3. 解决的痛点

### 供给端（AI 创作者/OPC/工作室）
- 有作品但缺曝光、无品牌背书
- 不会展示商业价值、不会报价
- 缺乏企业级信用、获客不稳定

### 需求端（企业/品牌方/中小商家）
- 想用 AI 但不知道找谁、花多少钱、怎么验收
- 无法评估服务商能力（技术黑盒）
- 缺乏标准验收流程，怕无休止返工

### 平台的价值
**把分散、无序、难验证的 AI 能力，转化成企业能理解、能采购、能验收的商业服务。**

---

## 4. 商业闭环

```
供给方入驻 → 平台筛选包装 → 需求方提交需求 → 平台匹配推荐 → 项目成交 → 交付评价沉淀 → 信任资产积累 → 吸引更多供需双方
```

### 三大信任基石
1. **里程碑式资金托管 (Escrow)**：按需求拆解/Demo/上线/交付四个节点释放资金
2. **标准化验收 (SLA)**：AI 项目专属验收白皮书，明确容错率和系统边界
3. **需求翻译与项目介入**：将模糊商业诉求转化为可执行的 PRD

### 收入节奏
| 阶段 | 模式 | 说明 |
|------|------|------|
| 第一阶段 | 线索与认证收入 | 服务商认证费、案例包装费、需求诊断费 |
| 第二阶段 | 撮合佣金 | 项目金额 5%-15%，小额固定服务费 |
| 第三阶段 | 企业会员 | 月度推荐、采购顾问、优先匹配 |
| 第四阶段 | 广告与品牌合作 | 月访问量 >10 万后启动 |

---

## 5. 服务商分级准入

| 等级 | 门槛 | 权益 |
|------|------|------|
| **L1 初级认证** | 实名 + ≥3 个 AI 商业作品 + 基础报价 | 基础信息流展示 |
| **L2 商业验证** | ≥1 个含 ROI 数据案例 + 48h 响应 SLA | 精选标识，推荐算法优先 |
| **L3 企业合伙人** | ≥5 单平台担保交易 + 4.8+ 评分 + 独立开票 + NDA | 1v1 顾问包装，优先高净值项目 |

---

## 6. 竞品格局

### 国内
- 即梦 AI：工具平台自带社区，不解决跨工具撮合
- 可灵 AI：工具+社区+创作者商业化，绑定自身视频生态（**最需关注的竞品**）
- LiblibAI / 吐司：偏模型和图像创作社区
- 站酷：泛设计平台，非 AI 原生，交付标准不足

### 国外
- Product Hunt：新品发布平台，不负责服务商撮合
- Futurepedia：AI 工具导航，偏索引
- Civitai / Hugging Face：偏开发者社区
- Behance/Dribbble：设计师作品集，企业采购属性弱

### 机会点
**跨工具、跨平台、面向企业采购、强调可信筛选和商业交付的 AI 服务商精选撮合平台目前是空白。**

---

## 7. 技术栈

| 层 | 选型 | 原因 |
|----|------|------|
| 前端 | Next.js (React) / Vue 3 | SSR 利于 SEO，生态成熟 |
| 后端 | Node.js (Express/NestJS) | 适配国内内容安全 API 中转，异步并发友好 |
| 数据库 | PostgreSQL + Redis | JSONB 支持非标表单，pgvector 支持后期 AI 检索 |
| 云服务 | 阿里云 / 腾讯云 | 国内合规（ICP 备案、内容审核组件） |
| 部署 | 单体 + Serverless | 主站单体保证迭代速度，AI 接口剥离至云函数 |

---

## 8. 合规红线

- **必须国内服务器**，完成 ICP 备案与公安联网备案
- 涉及资金担保需规划 **EDI 许可证**
- 双向内容安全隔离：国内模型 API 调用必须经内容审核中转，禁止直连海外模型
- 依据《AI 生成合成内容标识办法》，所有案例/图片/视频必须带 AI 标识
- 服务商须声明模型与数据集版权合法性，承担版权瑕疵担保责任

## 10. 启动路线图

| 时间 | 目标 |
|------|------|
| **第 0 个月** | 创始人亲自当「001 号服务商」，跑通 1 个真实企业项目（1-5 万），打磨合同和验收标准 |
| **第 1-2 个月** | 上线 Demo 官网，展示 10-20 个样板服务商，开放需求入口 |
| **第 3-6 个月** | 招募 30-50 个真实服务商，不收供方费用，靠需求诊断费+托管抽佣造血 |
| **第 6 个月后** | 上线认证/撮合佣金/企业会员 |
| **第 12 个月后** | 行业榜单、Demo Day、认证培训、标准化服务包 |

## 11. 推广策略

- **个人 IP 必须做**，做「判断型内容」而非泛 AI 热点
- 广告不要一开始重投（信任型产品不能靠流量硬灌）
- 内容平台优先级：小红书（种草+线索） > B站（深度+权威） > 公众号（沉淀+私域）
- 广告放大标准：企业线索成本 < 100-300 元，有效需求率 > 20%

---

## 12. 推广策略

- **第一版不是完整平台，而是「信任型 Demo」**
- 快速做官网 + 飞书多维表格后台 + 企业微信承接线索
- 核心目标：让别人看到你已经在做这件事

---

## 15. 完整信息架构 (IA)

### 页面总览（v1.0 上线范围）

| # | 页面 | 路由 | 优先级 | 类型 | 数据来源 |
|---|------|------|--------|------|----------|

| 2 | 服务商列表 | `/providers` | P0 | 列表+筛选 | 飞书多维表格 |
| 3 | 服务商详情 | `/providers/:slug` | P0 | 详情页 | 飞书多维表格 |
| 4 | 案例库 | `/cases` | P0 | 列表 | 飞书多维表格 |
| 5 | 案例详情 | `/cases/:slug` | P0 | 长文 | 飞书多维表格 |
| 6 | 发布需求 | `/post-demand` | P0 | 表单 | 飞书表单 → 表格 |
| 7 | 服务商入驻 | `/join` | P0 | 表单 | 飞书表单 → 表格 |
| 8 | 验收标准 | `/sla` | P1 | 静态长文 | 硬编码 |
| 9 | 用户协议 | `/terms` | P1 | 静态 | 硬编码 |
| 10 | 隐私政策 | `/privacy` | P1 | 静态 | 硬编码 |
| 11 | 服务商协议 | `/provider-agreement` | P1 | 静态 | 硬编码 |

### 页面详细设计

#### 1. 首页 `/` 【已做 Demo】
- Hero（痛点标题 + 双 CTA + 信任数字条）
- 三步流程说明
- 精选服务商卡片（3-6 张）
- 案例深度展示（2 个编辑精选）
- 信任基础设施（资金托管 / SLA / 认证）
- 双端入口（企业需求 / 创作者入驻）
- 底部 CTA（需求诊断）

#### 2. 服务商列表 `/providers`
- **筛选维度**：认证等级（L1/L2/L3）、擅长行业、预算区间、交付周期、评分
- **排序**：默认按认证等级+评分降序
- **卡片内容**：名称、等级徽章、标签、行业、周期、预算、评分、案例数
- **交互**：点击进入详情页

#### 3. 服务商详情 `/providers/:slug`
- 基本信息（名称、等级、认证时间）
- 擅长方向 + 技术栈标签
- 代表案例列表（关联案例库）
- 报价区间 + 交付周期
- 可交付服务清单
- 客户评价（评分+文字）
- 联系/咨询 CTA

#### 4. 案例库 `/cases`
- **筛选维度**：行业、服务类型、预算区间
- **列表卡片**：标题、服务商、行业、ROI 摘要、预算
- **交互**：点击进入详情页

#### 5. 案例详情 `/cases/:slug`
- 完整叙事结构：企业背景 → 痛点 → 方案拆解 → 交付过程 → 可量化效果 → 预算
- 服务商信息
- AI 标识（合规强制）
- 「索取此方案报价」CTA

#### 6. 发布需求 `/post-demand`
- **表单字段**：姓名、公司、行业（下拉）、业务痛点（多行文本）、预算区间（分档选择）、期望交付时间、是否需要平台推荐、联系方式
- 提交后 → 飞书表格 + 企业微信通知
- 提交后页面：预计 24h 内产品经理联系

#### 7. 服务商入驻 `/join`
- **表单字段**：团队/个人名称、擅长方向（多选）、已有案例链接（3+）、技术栈、可接预算区间、是否可开票、联系方式
- 提交后页面：审核周期 3 个工作日内邮件通知
- 文案强调「申请成为精选服务商」，不写免费入驻

### 用户路径

**企业用户**：首页 → 浏览服务商/案例 → 发布需求 → 产品经理对接 → 签约 → 托管交付

**创作者用户**：首页 → 浏览案例（了解平台标准）→ 申请入驻 → 审核 → 获认证 → 接单

### 技术实现策略（v1.0）

- **纯静态站**：Next.js SSG，所有动态数据从飞书多维表格 API 拉取（构建时 or ISR）
- **表单**：飞书表单嵌入或用飞书 API 提交
- **无数据库**：v1.0 不上 PostgreSQL，所有内容管理走飞书
- **部署**：Vercel（快）+ 国内 CDN（合规待评估）或阿里云 OSS 静态托管

## 16. Logo 设计方案

### 设计方向

**核心意象**：「九章」（算法/系统/中国古代数学）+「甄选」（筛选/品质/印章认证）

### 方案 A：印章认证（推荐）

正方形印章式 Logo，中心为「甄」字，外框融入几何线条暗示算法网格。

```
┌──────────────┐
│  ╔══╗  ╔══╗  │
│  ║  ║  ║  ║  │  ← 四角几何块（算法网格）
│  ╚══╝  ╚══╝  │
│              │
│    甄        │  ← 「甄」字（甄选/认证）
│              │
│  ╔══╗  ╔══╗  │
│  ║  ║  ║  ║  │
│  ╚══╝  ╚══╝  │
└──────────────┘
```

- 暖铜色（#C2410C）印章底 + 米色留白
- 正方形/圆形均可，做印章感
- 适合 Favicon、APP 图标、名片

### 方案 B：几何筛选

放大镜 + 网格交叉，代表「从杂乱中找到对的人」。

```
  ◯ （圆形放大镜轮廓）
 /|\  内部 3×3 网格
/ | \ 其中一个节点高亮
```

- 更现代、更国际化
- 适合后期出海复用

### 推荐

**方案 A（印章认证）**更适合中国市场和「甄选」的品牌心智。方案 B 可留作国际版 Logo。
